# Тестовое задание: Верстка и фронтенд-разработка

Выполненное тестовое задание по макетам «Тестовые задания 1.0.1».
В основе проекта — семантическая разметка, оптимизация производительности и кроссбраузерная совместимость.


## Технологический стек
- **HTML** Семантическая разметка, нативная валидация форм.
- **CSS** Flexbox, CSS Grid, CSS-переменные, Scroll Snap, кастомные скроллбары, анимации.
- **JavaScript** Используется минимально и только там, где это необходимо (например, переключение темы с сохранением в `localStorage`).
- **Инструменты** Prettier, Stylelint (для обеспечения чистоты кода и отсутствия лишних пробелов/табуляций).

## Старт:
Откройте файл `index.html` в браузере.

## Реализованные задачи

### 1. Произвольный текст (Семантика и типографика)
- Использован широкий спектр семантических тегов: `<article>`, `<section>`, `<time>`, `<mark>`, `<details>`, `<figure>`, `<blockquote>`.
- Реализована "тёмная тема" через CSS-переменные (`:root` и `[data-theme="dark"]`) с сохранением выбора пользователя в `localStorage`.
- Адаптивная сетка на CSS Grid для корректного отображения на мобильных устройствах.

### 2. Форма обратной связи
- Доступность. Все `<label>` жестко связаны с `<input>` через `for`/`id`.
- Нативная валидация. Применение атрибутов `required`, `autocomplete`, `type="color"` и др.
- Состояния. `:focus-visible` для навигации, плавные переходы (`transition`) при взаимодействии.
- Карточка формы использует сложную многослойную тень (`box-shadow`), создающую эффект глубины и объема.

### 3. UI Кнопки (Модульность)
- Архитектура на базе модификаторов: базовый класс `.btn` отвечает за геометрию, модификаторы (`.btn-red`, `.btn-outline`) — только за цвета.
- Иконки реализованы через инлайн-SVG с `fill: currentColor` для автоматического наследования цвета текста.
- Стрелка в кнопке сделана чистым CSS (border-trick), без использования дополнительных изображений.
- Корректная обработка состояния `disabled` (отключение hover-эффектов, изменение курсора).

### 4. Схема связей
- Элементы размещены с помощью `position: absolute` относительно родительского контейнера `(position: relative)`
- Связи между узлами реализованы через встроенный векторный формат SVG (теги `<line>` и `<polyline>`), что обеспечивает идеальную четкость отрисовки линий при любом масштабе.
- - Для обозначения направления связей используются SVG-маркеры (`<marker>` с атрибутом `marker-start="url(#arrow)"`), которые автоматически отрисовывают аккуратные стрелки на линиях.
- Управление слоями через `z-index`: текст всегда находится поверх линий.

### 5. Витрина товаров (Слайдер)
- Горизонтальная прокрутка реализована с помощью `overflow-x: auto` и `scroll-behavior: smooth` для плавного перемещения.
- Для предотвращения сдвигов макета (CLS) и корректного отображения картинок, контейнеру изображения задана фиксированная высота (`300px`), а самим изображениям — свойства `max-width: 100%`, `max-height: 100%` и `object-fit: contain`.
- Добавлены градиентные маски (`::before` и `::after`) по краям слайдера для эффекта плавного визуального затухания контента.
- Реализован кастомный стиль полосы прокрутки (скроллбара) с закругленными краями и цветом.

### 6. Черная страница
1) Долгая загрузка
Блокирующий CSS в самом конце: Тег <link> для подключения стилей Bootstrap заброшен в самый низ файла (после закрывающего тега </html>).
Скрипты в <head> без defer — блокируют отрисовку страницы.
Браузер сначала строит дерево элементов (DOM), пытается всё отобразить без стилей, натыкается на CSS в конце и начинает перерисовывать страницу заново.

2) Страница грузится дольше
Тяжёлое фоновое изображение: В теге <html> прописана картинка .bmp. Формат BMP не сжимается и весит в десятки раз больше JPEG/PNG.

3) Страница «странно выглядит»?
`display: grit !important !important` — Опечатка в названии свойства grit (grid), а двойной !important полностью ломает CSS-правило. К тому же селектор * * * пытается применить это ко всем элементам с тройной вложенностью.
`position: absolute; top 10px` — Пропущено двоеточие после top.
`style="display: inline=block` — Опечатка, вместо знака = должен быть :.
`clas="product-device... `— Опечатка в названии атрибута class.
`style="background: transparant` — Опечатка в слове transparent.
`style="colour: red, text-align: centner;...` — Опечатка в названии свойства colour (color), вместо запятой должна быть точка с запятой, а в слове center опечатка.
`displat: -o-inline-grid `- Опечатка в названии свойства displat (display)
Навигация и футер закомментированы — часть контента скрыта
Плагины Bootstrap подключены дважды (один раз в <head>, второй — в конце перед </html>). Это не критично, но создаёт дублирование.

4) Ошибки в тексте
В описании: "постораться" (правильно: постараться), "забыт" (забыть), "увидете" (увидите), "зедсь" (здесь).

5) Пропажа
В закомментированной навигации спрятана предсмертная записка: «я больше так не могу, эта страница захавала мой мозг, тут слишком много всякой фигни,
 исправить которую у меня уже нет сил... прощайте, я в отпуск»

### 7. Красота
Реализованы сложные UI-элементы с реалистичным объемом и плавными анимациями в стиле скевоморфизм.

**Кнопка:**
- Градиент. Использовано 4 точки цвета в `linear-gradient` для создания эффекта объема (свет сверху, тень снизу).
- Комбинированные тени. `inset box-shadow` для внутреннего свечения + внешние тени для глубины.
- Объемный текст. `text-shadow` для создания рельефности текста.
- Нажатие. `transform: translateY(2px)` + изменение теней при `:active` создают ощущение физического нажатия.
- Состояния. normal, hover, active — все анимированы через `transition: all 0.1s ease`.

**Объемный переключатель:**
- Объемный шарик отрисовывается на Canvas через JavaScript (SphereModel + SphereView), что позволяет создавать реалистичный 3D-эффект с градиентами и бликами программно, а не через CSS.
- Приложение разделено на три слоя:
  * Model (SphereModel.js) — хранит состояние и угол поворота сферы
  * View (SphereView.js) — отвечает за отрисовку сферы на canvas
  * Controller (SwitchController.js) — управляет взаимодействием с пользователем и координирует Model/View
